## Multi-pass Phi word creation process:

Pass 1: Check that the gloss we are considering is not already in the lexicon
  a. Confirm with user

Pass 2: Create the gloss, the concept, and a detailed description
  a. Present to user
  b. Wait for approval
  c. Insert into JSON if approved

Pass 3: Create the Phi word and develop a poetic sound symbolism
  a. Present to user
  b. Wait for approval

Pass 4: Check that the Phi word is not already in the lexicon
  a. Insert into JSON if available

Pass 5: Analyze POS with cross-linguistic examples
  a. Do a deep analysis
  b. Present to user
  c. Wait for approval
  d. Insert into JSON if approved

Pass 6: Determine pillars through deep analysis, adding rationales
  a. Do a deep analysis
  b. Present to user
  c. Wait for approval
  d. Insert into JSON if approved

Pass 7. Assign tags: analyze which semantic domains genuinely apply (not forcing any number)
  a. Do a deep analysis
  b. Present to user
  c. Wait for approval
  d. Insert into JSON if approved

Pass 8: Check that the JSON formtting is in the preferred format
  a. JSON _must_ match schema.json
  b. Make sure the word defeinition is coherent


## Use a template/checklist for consistency:

  - [ ] Phi word follows phonological rules
  - [ ] Sound symbolism connects to meaning
  - [ ] POS verified cross-linguistically
  - [ ] Pillars deeply analyzed
  - [ ] Concept clearly defined
  - [ ] Description captures nuance
