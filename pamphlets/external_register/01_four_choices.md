# Part 1: The available choices

Before reaching outside the lexicon, ask what the conversation actually needs.

## Core composition

Use ordinary Phi when its components expose the analysis you want. The Ring Verse's *lord* can be discussed as `rena pilu miona`, the person who takes, when the act matters more than the inherited title.

```
rena pilu miona to shua.
REL take person PST come.
(The person who takes came.)
```

## Registered compound

Use an established compound when the community has already stabilized a transparent Phi expression. Compounds remain made of core words and require no external boundary.

## Productive Phi-form name

When a bearer accepts one lowercase, two-, three-, or four-syllable Phi-shaped token as their name, `ne` licenses it directly. It is not a dictionary word and gains no lexical meaning.

```
ne samira so shua.
NAME samira FUT come.
(samira will come.)
```

## Adapted guest

Use `hasha … hasho` when a five-or-more-syllable, multi-token, or explicitly external name or term should be pronounceable and renderable in Phi, but should not be mistaken for shared core vocabulary.

```
ne hasha samira kaleno hasho so shua.
NAME EXT.GUEST [samira kaleno] EXT.GUEST.CLOSE FUT come.
(Samira Kaleno will come.)
```

## Exact opaque material

Use `patha … patho` when spelling, script, punctuation, notation, identity wording, or numerical precision is part of what must survive.

```
ne patha 李明 patho so shua.
NAME EXT.EXACT [李明] EXT.EXACT.CLOSE FUT come.
(Li Ming will come.)
```

The same referent may have an exact form, an agreed guest form, and an agreed single-token onym. Each occurrence chooses one construction. Guest and exact payloads always keep their boundaries; a productive onym is not an elided guest payload but a separate name atom licensed directly by `ne`.
