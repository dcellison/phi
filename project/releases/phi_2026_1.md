# Phi 2026.1

**Status:** ready for maintainer review

## Purpose

Phi 2026.1 makes the language's scope honest and gives it a bounded way to carry names, quotations, technical precision, and analytically necessary concepts from outside the core lexicon.

## Language changes

- Added `hasha … hasho`, the adapted guest frame.
- Added `patha … patho`, the exact opaque frame.
- Limited core spelling, punctuation, and mode-invariance rules to core and guest material while preserving Phi boundary words in every mode.
- Clarified that core refusals do not prevent faithful marked reporting.
- Recast modifier-first as an organizing principle and deliberate pace as a reference practice rather than a physical impossibility.

## Tooling changes

- The validator recognizes balanced external frames, validates guest phonotactics, ignores exact payload internally, checks the doubled closer escape, and continues enforcing all core rules outside the frames.
- Generated pages HTML-escape external payload and visually distinguish guest and exact material.
- Tengwar output renders guest material and preserves exact payload in its source form between rendered Phi boundaries.
- Automated regression tests cover valid, malformed, escaped, opaque, and injection-like payloads.

## Compatibility

No existing core word or construction is removed or renamed. Earlier claims that all Phi material is lowercase, period-only, or recoverable letter-for-letter remain true for core Phi but not for exact external payload. Existing core names remain unchanged; people may now retain an external name by framing it on every occurrence.
