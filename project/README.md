# Phi project records

This directory records how Phi is maintained. The index below gathers its working records and release manifests. These records govern the work around the language. [`canon.md`](../canon.md), the lexicon, and the grammar references remain authoritative for the language itself.

| Resource | Purpose |
|---|---|
| [Development protocol](development_protocol.md) | Quality, evidence, and validation requirements for language changes. |
| [Development log](development_log.md) | Lightweight record of proposed, accepted, superseded, and rejected decisions. |
| [Status roadmap](roadmap.md) | Current workstreams, dependencies, evidence gates, and solo work packages. |
| [Deferred questions](deferred_questions.md) | Parked questions and decisions, with the condition for returning to each one. |
| [Publishing strategy](publishing.md) | Order of operations for public access and eventual print publication. |
| [Release manifests](releases/) | Versioned records of what each release contained and claimed at the time. |

Release manifests preserve their original scope even when later decisions supersede a feature. Historical studies and retired working material live in [`archive/`](../archive/).
